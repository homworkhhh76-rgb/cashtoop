# Revision 39 — تأمين MongoDB بعد تغيير كلمة المرور

## ما تم تغييره

- أضيفت نقطة MongoDB Serverless داخل نفس المشروع: `api/rtdb.js`.
- لم يتم وضع `MONGODB_URI` أو اسم مستخدم MongoDB أو كلمة المرور داخل HTML/JS أو داخل إعدادات الواجهة.
- الواجهة المنشورة عبر HTTPS تستخدم `/api/rtdb` من نفس المشروع أولاً، مع إبقاء نقطة الاتصال القديمة كـ fallback للتوافق فقط.
- فتح المشروع مباشرة عبر `file://` يستمر باستخدام API القديم لأن `file://` لا يستطيع تشغيل Serverless API محلياً.
- Service Worker يستثني `/api/rtdb` من الكاش بالكامل حتى لا تُخزن استجابات قاعدة البيانات الحية.
- تم تحديث Cache Storage إلى Revision 39 مع استمرار Cache First للصفحات والملفات المحلية.

## أين توضع كلمة مرور MongoDB الجديدة؟

ضع رابط الاتصال الكامل في متغير بيئة سري باسم `MONGODB_URI` داخل لوحة الاستضافة، وليس داخل أي ملف في المشروع.

المتغيرات المطلوبة:

- `MONGODB_URI`: رابط الاتصال الكامل بعد تحديث كلمة المرور.
- `MONGODB_DB`: **نفس اسم قاعدة البيانات القديمة** حتى تبقى البيانات القديمة كما هي.
- `MONGODB_COLLECTION`: اسم مجموعة التخزين. القيمة الافتراضية في النسخة الجديدة `cashtop_rtdb`، ويمكن ضبطها على اسم المجموعة المستخدمة في API القديم إن كانت مختلفة.
- `CASHTOP_ALLOWED_ORIGINS`: نطاق أو نطاقات التطبيق مفصولة بفواصل عند الحاجة.

بعد تعديل Environment Variables يجب إعادة نشر Deployment جديد حتى تقرأ الدالة كلمة المرور الجديدة.

## مهم جداً

لا يوجد تشفير آمن لكلمة مرور قاعدة البيانات إذا وضعت كلمة التشفير ومفتاح فك التشفير داخل نفس ملفات الواجهة. الحل الآمن هو إبقاء السر في Environment Variables الخاصة بالخادم. منصات الاستضافة تحفظ هذه القيم بعيداً عن JavaScript الذي يصل للمستخدم.

بما أن كلمة المرور الجديدة تمت مشاركتها نصياً أثناء العمل، يفضّل تدويرها مرة أخرى قبل الاستخدام الإنتاجي النهائي، ثم وضع القيمة النهائية مباشرة في Secrets/Environment Variables فقط.

## الحفاظ على البيانات القديمة

تغيير كلمة مرور مستخدم MongoDB لا يحذف البيانات. للحفاظ على نفس البيانات يجب أن يشير `MONGODB_URI` إلى نفس Cluster ونفس المستخدم (بعد تحديث كلمة مروره)، وأن يبقى `MONGODB_DB` على نفس اسم قاعدة البيانات السابقة.

طبقة `api/rtdb.js` تستخدم تخزيناً متوافقاً مع المسارات، وتقرأ أيضاً أنماطاً قديمة شائعة (`payload/data/value/tree`) وتطبق تحديثات المسارات الجديدة كطبقة overlay حتى لا تحتاج إلى حذف البيانات القديمة أثناء الانتقال.
