# Revision 40 — إصلاح المزامنة بعد تغيير كلمة مرور MongoDB

- إعادة `https://cash-top-api-2026.vercel.app/api/rtdb` ليكون نقطة المزامنة الأساسية حتى تعمل الاستضافات الثابتة وفتح `file://`.
- إبقاء `/api/rtdb` داخل نفس المشروع كخطة احتياطية فقط عندما تدعم الاستضافة Serverless Functions.
- إصلاح حالة كانت فيها بعض الاستضافات تعيد `index.html` بحالة HTTP 200 لمسار `/api/rtdb`؛ الآن لا تعتبر استجابة ناجحة ويتم الانتقال تلقائياً إلى API التالي.
- تطبيق نفس التحقق على المزامنة وتسجيل الدخول ولوحة الإدارة.
- تحديث Cache Storage إلى `v50-r40-sync-api-fallback-cache-first`.
- تنبيه مهم: إذا كانت كلمة مرور MongoDB تحتوي على `@` فيجب ترميزها داخل `MONGODB_URI` إلى `%40`.
- يجب تحديث `MONGODB_URI` في مشروع Vercel الذي يشغل `cash-top-api-2026` ثم تنفيذ Redeploy. لا توضع كلمة المرور في ملفات الواجهة.
