# Revision 41 — ربط API الجديد + فتح الصفحات فوراً من الكاش

- جعل `/api/rtdb` من نفس نطاق التطبيق هو نقطة المزامنة الأولى عند تشغيل CASH TOP على استضافة تدعم الـ API، وهذا يمنع الاعتماد على CORS أو نطاق خارجي عندما يكون التطبيق والـ API في المشروع نفسه.
- إضافة رابط الـ API الجديد الذي تم اختباره بنجاح كخطة أساسية عند عدم وجود API محلي:
  - `https://cashtop-2bblxfx4i-cashtop.vercel.app/api/rtdb`
- إضافة Alias فرع `main` كخطة احتياطية، ثم إبقاء API القديم كخطة احتياطية أخيرة فقط.
- لا توجد كلمة مرور MongoDB أو Connection String داخل ملفات الواجهة؛ بيانات MongoDB تبقى داخل Environment Variables على Vercel.
- تحديث Cache Storage إلى `v51-r41-new-api-instant-pages-cache-first`.
- جميع صفحات وملفات التطبيق المحلية يتم تخزينها مسبقاً عند تثبيت Service Worker.
- أي صفحة موجودة في الكاش تُفتح من Cache Storage فوراً حتى لو كان الإنترنت متوفراً؛ الشبكة لا تؤخر فتح الصفحة.
- تقليل تحديث HTML في الخلفية إلى مرة كل 10 دقائق، والملفات الثابتة إلى مرة كل 30 دقيقة، لتقليل منافسة الشبكة مع واجهة المستخدم.
- إضافة بحث إضافي داخل الكاش حسب pathname لطلبات التنقل، حتى لو تغيّرت query parameters أو صيغة الرابط.
- تبقى طلبات `/api/rtdb` وبيانات المزامنة خارج Cache Storage دائماً حتى لا تُقرأ بيانات قديمة بين الأجهزة.
