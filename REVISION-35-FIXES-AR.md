# CASH TOP 2 — Revision 35

## إصلاح ظهور «تعذر الاتصال» رغم وجود الإنترنت

تم تعديل طبقة المزامنة بحيث لا تعتمد على `navigator.onLine` كحكم نهائي على وجود الاتصال؛ بعض أنظمة التشغيل وبيئات VPN/Proxy تعطي قيمة غير دقيقة. أصبحت المزامنة تحاول خادم قاعدة البيانات فعلياً وتحدد الحالة من نتيجة الطلب.

### إصلاح CORS المسبب لـ Failed to fetch

في وضع MongoDB API كانت طلبات GET/PUT ترسل رؤوساً مثل `Cache-Control` و`Pragma` و`X-Firebase-ETag`. هذه الرؤوس قد تجبر المتصفح على CORS preflight أو تجعل الخادم يرفض `Access-Control-Allow-Headers`، فتظهر `Failed to fetch` مع أن الإنترنت يعمل. في Revision 35 يتم حذف هذه الرؤوس غير الضرورية عند الاتصال بـ MongoDB API، مع إبقاء `cache: no-store` داخل Fetch لمنع كاش بيانات القاعدة.

## تحسين متانة الاتصال

- إعادة محاولة الطلبات المؤقتة تلقائياً عند 408/425/429/500/502/503/504.
- محاولة نقطة `/api/rtdb` على نفس نطاق التطبيق كمسار احتياطي عندما يكون API الخارجي غير متاح وكان التطبيق منشوراً مع الـ API في نفس المشروع.
- عدم إيقاف المزامنة أو سحب البيانات بسبب قيمة `navigator.onLine` وحدها.
- العمليات المعلقة تستمر في إعادة المحاولة تلقائياً دون حذفها.
- تسجيل حالة وصول الخادم الفعلية لاستخدامها في مؤشر الاتصال.
- صفحة الدخول تحاول قاعدة البيانات مباشرة ثم تستخدم البيانات المحلية كخطة احتياطية عند فشل الخادم.
- لوحة الإدارة لم تعد تمنع مزامنة الشركات اعتماداً على `navigator.onLine`، وتم تنظيف رؤوس CORS غير الضرورية في طلبات MongoDB API.

## تحديث الكاش

- إصدار Cache Storage جديد: `v45-r35-backend-retry-cache-first`.
- جميع صفحات وملفات التطبيق المحلية تُحمّل إلى الكاش أثناء تثبيت Service Worker.
- فتح الصفحات والملفات المحلية أصبح Cache First حتى مع توفر الإنترنت.
- HTML يُحدّث في الخلفية بوتيرة أسرع، وملفات JS/CSS والصور تُحدّث في الخلفية دون تعطيل فتح الصفحة.
- عند رسالة WARM_CACHE يتم تحديث حزمة التطبيق المحلية كاملة في الخلفية، وليس فقط التحقق من الملفات المفقودة.
- يتم طلب تحديث Service Worker عند فتح التطبيق مع `updateViaCache: none`.

## ملاحظة تشغيلية

إذا كانت نقطة MongoDB API نفسها متوقفة أو إعداد CORS على الخادم يمنع `PUT` بالكامل، فلن تستطيع الواجهة تجاوز توقف الخادم. النسخة الحالية تعالج أخطاء CORS الناتجة عن الرؤوس الإضافية، وتضيف إعادة المحاولة والمسار الاحتياطي وتحافظ على العمليات محلياً إلى أن يستجيب الخادم.
