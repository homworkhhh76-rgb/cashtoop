# Revision 54 — سجل 100 عملية وإشعارات Push وإزالة الصيانة والنوافذ الفورية

- سجل النشاط مناسب للجوال مع تمرير أفقي كامل وتمرير رأسي لا يحبس الصفحة عند الوصول للحواف.
- الاحتفاظ بآخر 100 عملية تدقيق فقط محلياً وعلى مسار `auditTrailRecent`، مع حذف الأقدم تلقائياً وتنظيف سجل التدقيق القديم عند أول مزامنة ناجحة.
- حذف وضع الصيانة وصفحة الصيانة وكل تحويلات الدخول المرتبطة بها.
- حذف نظام النوافذ الفورية للمستخدمين والعملاء وإيقاف مزامنته.
- إشعارات النظام مغلقة افتراضياً وتظهر للمدير فقط عند التفعيل من إعدادات النظام.
- إشعار فاتورة جديدة، تنبيهات المخزون والتنبيهات الذكية، وملخص المبيعات والأرباح الساعة 11 مساءً مع زر عرض الملخص فوراً.
- استخدام `notification-icon.png` كأيقونة الإشعارات.
- دعم `notificationclick` لفتح الصفحة المناسبة، وWeb Push للخلفية عند تشغيل خدمة Push المرفقة وضبط VAPID.
- صفحة `admin-notifications.html` لإرسال إشعار مخصص بعنوان ونص وروابط الأيقونة والصورة والوجهة. محتوى الإشعار لا يُخزن؛ خدمة Push تخزن اشتراكات الأجهزة فقط.
- تحديث Service Worker إلى `v64-r54-audit100-notifications-no-maintenance-cache-first` مع Cache First للجوال واللابتوب.

## ملاحظة تقنية مهمة
الإشعار العام الذي يصل إلى أجهزة مغلقة أو مستخدمين غير مسجلين يحتاج Web Push حقيقياً. لذلك أضيفت واجهة العميل وService Worker ومثال خادم `push-api-example`. يلزم نشر مسارات Push على خادم HTTPS وضبط مفاتيح VAPID؛ من دون خادم Push تعمل إشعارات المدير المحلية أثناء تشغيل التطبيق، لكن لا يمكن للمتصفح وحده إرسال إشعار عالمي لأجهزة أخرى وهي مغلقة.
